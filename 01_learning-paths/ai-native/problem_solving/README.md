## ğŸ’¡ Quick Reference

### The Problem Definition Canvas

Before diving into ANY project, fill this out (takes 15 minutes):

```markdown
# [Project Name]

## Current State
(What's the problem right now?)

## Desired State
(What success looks like)

## Success Criteria
(How you'll know it works)

## Constraints
(What you CAN'T do)

## Core Features
(What absolutely must work)

## Nice-to-Haves
(What would be cool but isn't essential)

## Edge Cases
(What could go wrong?)
```
